## What is a topic?

A **topic** is a named channel in Kafka. Agents **publish** to a topic; other agents **subscribe** to receive messages from that topic.

Topics decouple agents: the Scout does not need to know the Analyst exists — it only publishes to `nexus.research`.

## Default topics

Nexus creates these topics on startup:

| Topic | Purpose |
| --- | --- |
| `nexus.research` | Raw research / headlines from Scout |
| `nexus.analysis` | Ranked analysis from Analyst |
| `nexus.writing` | Draft content from Writer |
| `nexus.delivery` | Delivery confirmation from Deliverer |
| `nexus.deadletter` | Failed messages after retries |
| `nexus.heartbeat` | System heartbeat events |

Default topics **cannot be deleted** from the dashboard.

## Demo pipeline flow

```
Scout        → nexus.research
Analyst      → nexus.analysis
Writer       → nexus.writing
Deliverer    → nexus.delivery
```

## Custom topics

Admins can create topics from the dashboard or API:

```
POST /api/v1/topics
```

```json
{
  "name": "nexus.custom",
  "partitions": 1,
  "retention_days": 7
}
```

Custom topic names **must start with** `nexus.`.

List topics with `GET /api/v1/topics`. The dashboard **Topics** page shows partition count, message count, and payload size from Redis.

## Naming convention

Use the `nexus.<stage>` pattern so topics are easy to identify and filter:

- `nexus.ingest` — raw input
- `nexus.review` — human or QA step
- `nexus.notify` — outbound notifications
