## What is a message?

A **message** is a packet of data one agent sends through the bus. Every message includes:

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique message ID (`msg_...`) |
| `correlation_id` | string | Pipeline run this message belongs to |
| `topic` | string | Kafka topic the message was published to |
| `sender_agent` | string | Name of the publishing agent |
| `payload` | object | Arbitrary JSON content |
| `timestamp` | ISO datetime | When the message was created |
| `retry_count` | integer | Retry attempts (for DLQ handling) |

Example:

```json
{
  "id": "msg_a1b2c3d4",
  "correlation_id": "run_1780068737_a286ob",
  "topic": "nexus.research",
  "sender_agent": "scout-agent",
  "payload": { "status": "research_done", "count": 10 },
  "timestamp": "2026-05-29T21:02:20.123456+00:00",
  "retry_count": 0
}
```

## Publishing

Agents publish with `POST /api/v1/messages/publish`:

```json
{
  "topic": "nexus.research",
  "correlation_id": "run_abc123",
  "sender_agent": "scout-agent",
  "payload": { "status": "research_done" }
}
```

Nexus validates the topic exists, records the message in Redis (7-day TTL), updates the pipeline tracker, increments the sender's message count, and broadcasts `message.published` to the dashboard.

## Consuming

The v0.1 SDK polls `GET /api/v1/messages?topic=<topic>&limit=N` for new messages on the agent's subscribed topic. Messages the agent has already seen are skipped.

Register a handler with `on_message` / `onMessage`, then call `start()`.

## Message history

The dashboard **Live Messages** page and `GET /api/v1/messages` show recent messages. Filter by:

- `topic` — messages on a specific topic
- `agent` — messages from a specific sender

Retrieve one message with `GET /api/v1/messages/{id}`.

## Correlation IDs

Every message in a pipeline run shares the same `correlation_id`. This links messages, context keys, and pipeline history together. Generate one per run (e.g. `run_<timestamp>_<suffix>`).
