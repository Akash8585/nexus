## What is the context store?

The **context store** is a Redis-backed key-value store where agents read and write shared data for a pipeline run. Instead of embedding large payloads in every message, agents pass lightweight signals on topics and store bulky data in context.

Example from the demo:

1. Scout writes `raw_headlines` to context
2. Analyst reads `raw_headlines`, writes `top5_stories`
3. Writer reads `top5_stories`, writes `final_briefing`
4. Deliverer reads `final_briefing`, writes `delivery_status`

## Keys and correlation IDs

Context keys are scoped by **correlation ID**:

```
context/{correlation_id}/{key}
```

Example:

```
context/run_1780068737_a286ob/top5_stories
context/run_1780068737_a286ob/final_briefing
```

Every message in the run uses the same `correlation_id`, so all agents share one namespace.

## TTL

Context values expire after **24 hours** by default. Pass `ttl_hours` when writing via the API or SDK.

Admins can flush all context from **Settings** (destructive — dev/test only).

## API

| Method | Endpoint | Description |
| --- | --- | --- |
| GET | `/api/v1/context/{correlation_id}` | All keys for a run |
| GET | `/api/v1/context/{correlation_id}/{key}` | One value |
| PUT | `/api/v1/context/{correlation_id}/{key}` | Write a value |

Writes require an **API key**. Reads accept API key or JWT.

## SDK

<CodeGroup>

```python Python
agent.context.get("run_abc", "top5_stories")
agent.context.set("run_abc", "draft", {"content": "..."})
agent.context.get_all("run_abc")
```

```typescript JavaScript
await agent.context.get("run_abc", "top5_stories");
await agent.context.set("run_abc", "draft", { content: "..." });
await agent.context.getAll("run_abc");
```

</CodeGroup>

## Dashboard

Open a pipeline run in **Pipelines → [correlation ID]** to view all context keys and replay messages step by step.
