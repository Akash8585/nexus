## What is Nexus?

Nexus is an open-source **multi-agent coordination bus**. It is the infrastructure layer between AI agents so they can communicate, share memory, and collaborate on tasks without being hardcoded to know about each other.

Think of Nexus as a postal system for AI agents. Every agent sends and receives messages through Nexus. Nexus handles delivery, persistence, routing, and observability. Agents focus only on their job.

> The coordination layer for multi-agent AI systems. Connect agents. Coordinate tasks. See everything.

## The problem

When developers build systems with more than one AI agent, the same problems appear every time:

- **No standard communication layer** — agents call each other directly, tightly coupled and hard to scale
- **No shared memory** — each agent starts fresh with no awareness of prior work
- **No fault tolerance** — if one agent crashes, in-flight work can be lost
- **No visibility** — no single place to see messages, agent health, or pipeline failures

Nexus provides a standard answer to all four.

## Key concepts

| Concept | Summary |
| --- | --- |
| **Agent** | A program that registers with Nexus, subscribes to a topic, and processes messages |
| **Topic** | A named Kafka channel (e.g. `nexus.research`) |
| **Message** | A payload with metadata: sender, topic, correlation ID, timestamp |
| **Correlation ID** | Unique ID for one pipeline run — ties messages and context together |
| **Context store** | Redis key-value store for shared pipeline data |
| **Pipeline** | A sequence of agents working toward one goal (e.g. Scout → Analyst → Writer → Deliverer) |
| **Dead letter queue** | Storage for messages that failed after retries |

Read the [Concepts](/concepts/agents) section for details on each area.

## Who is it for?

Nexus is built for **developers who have built AI agents and need those agents to work together reliably in production** — with observability, recovery, and a clear integration path.
