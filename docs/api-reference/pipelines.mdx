---
title: Pipelines
description: Start, complete, list, and rerun pipeline executions.
---

Base URL: `http://localhost:8000/api/v1/pipelines`

## POST `/pipelines/start`

Register a new pipeline run.

**Auth:** API key

```json
{
  "correlation_id": "run_abc123",
  "trigger_input": "Morning briefing on AI news"
}
```

**Response:** `200` — Pipeline run object with `status: running`. Broadcasts `pipeline.started` via WebSocket.

## POST `/pipelines/{correlation_id}/complete`

Mark a run completed or failed.

**Auth:** API key

```json
{ "status": "completed" }
```

`status` may be `completed` or `failed`. Broadcasts `pipeline.completed` or `pipeline.failed`.

## GET `/pipelines`

List pipeline runs.

**Auth:** JWT

**Query:** `status` (optional), `limit` (default `50`)

Stale runs with no activity auto-fail after 5 minutes.

## GET `/pipelines/{correlation_id}`

Get one run's details.

**Auth:** JWT

## POST `/pipelines/{correlation_id}/rerun`

Register a **new** run with the same trigger input. Does not execute agents — run `python demo/run.py` to execute.

**Auth:** Admin JWT

**Response:** `200` — `{ "correlation_id": "run_<new_uuid>" }`
